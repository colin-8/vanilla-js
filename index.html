General Knowledge base - Mozilla Developer Network (MDN)

This is the #1 reference for any JS developer from beginners till very experienced ones. Go though each section, each article, each example. Go though each Object, each API, each method, each parameter. I promise you will find a lot of small tricks or cool functions you never heard about.

For everyone who knows a bit of jQuery - You Might Not Need jQuery

This is a solid starting point for everyone who was working with jQuery. Today it is still very useful resource even for those who never used jQuery because it contains all the main JavaScript functions from AJAX till DOM manipulation in one place.

Videos

  • FunFunFunction - My favorite YouTube channel about learning JavaScript by MPJ. Even if you know a lot, watching his videos is funny.
  • JavaScript: The Good Parts by Douglas Crockford. The best way to learn is to see how others are coding. Go to GitHub and browse the source code of many JavaScript projects.

JavaScript design patterns

  • Learning JavaScript Design Patterns - A book by Addy Osmani, O'Reilly
  • JS patterns (http://shichuan.github.io/javascript-patterns/)

Metaprogramming in JavaScript

Awesome blog about how to program JavaScript itself. Want to change what 'instanceof' do or what happens if property on object doesn't exists?

Other great resources

  • JavaScript Scene (https://medium.com/javascript-scene) - probably #1 blog about modern JavaScript in the world, Eric Elliott. I strongly recommend to go though all the Eric Elliott's JavaScript articles like "Two Pillars of JavaScript", read and re-read them.
  • ②ality – JavaScript and more - great Dr. Axel Rauschmayer's blog about the JavaScript.
  • Eloquent JavaScript - second edition, book by Marijn Haverbeke

And many links available here:

  • Essential JavaScript links (https://github.com/ericelliott/essential-javascript-links) - a huge collection of links
  • Quora - What is the best way to learn JavaScript?

Community (Slack)

  • Frontend Developers - World's biggest Slack channel about the frontend.

Just open DevTools and start experimenting!

P.S. I also recommend to dig into the history of JavaScript, asynchronous programming, Web Standards, HTTP, UI development, DOM, events, accessibility and software engineering and architecture in general.
